Demo01: Render JSX（ReactDOM.render())

ReactDOM.render是基本方法，用于将模板转换成html语言，并插入指定DOM节点

ReactDOM.render(
  <h1>Hello, world!</h1>,
  document.getElementById('example')
);

​ 将h1的hello,world!插入到了example节点

Demo02: Use JavaScript in JSX(JSX语法)

​ demo01,直接把html语言写在了js中，不加任何引号，这是JSX语法，js和html混写。

基本规则：遇到html标签（以<开头），就用html规则解析；遇到代码块（以{开头），用js解析。

var names = ['Alice', 'Emily', 'Kate'];

ReactDOM.render(
  <div>
  {
    names.map(function (name) {
      return <div>Hello, {name}!</div>
    })
  }
  </div>,
  document.getElementById('example')
);

Demo04: Define a component（组件）

​ 将代码封装成组件，像插入html标签一样，在网页中插入这个组件。

React.createClass方法用于生成这个组件类

class HelloMessage extends React.Component {
  render() {
    return <h1>Hello {this.props.name}</h1>;
  }
}

ReactDOM.render(
  <HelloMessage name="Shirley" />,
  document.getElementById('example')
);

​ 上面代码，变量HelloMessage是组件类。模板插入</HelloMessage>会自动生成实例。所有组件类都必须有自己的render方法，用于输出组件。

​ 组件与原生html标签用法完全一致，可以加入任意属性。比如上面的<HelloMessage name="Shirley" />,就是HelloMessage加了个name属性；它的属性可以在组件类的this.props对象上获取，比如上面的name属性通过this.props.name获取。

​ 注意：组件类的第一个字母必须大写。另外，组件类智能包含一个顶层标签。下面的包含了两个顶层标签h1``p,所以错。

// wrong
class HelloMessage extends React.Component {
  render() {
    return <h1>
      Hello {this.props.name}
    </h1><p>
      some text
    </p>;
  }
}

// correct
class HelloMessage extends React.Component {
  render() {
    return <div>
      <h1>Hello {this.props.name}</h1>
      <p>some text</p>
    </div>;
  }
}

​ 注意：添加组件属性，class要写成className，for属性写成htmlFor。

​ 原因：class``for是js保留字。

Demo05: this.props.children（获取组件的所有子节点）

​ 下面的代码NotesList有两个span子节点，可以通过this.props.children读取，运行结果：

1.hello

2.world

class NotesList extends React.Component {
  render() {
    return (
      <ol>
      {
        React.Children.map(this.props.children, function (child) {
          return <li>{child}</li>;
        })
      }
      </ol>
    );
  }
}

ReactDOM.render(
  <NotesList>
    <span>hello</span>
    <span>world</span>
  </NotesList>,
  document.getElementById('example')
);

​ 注意：this.props.children值（数据类型）有三种可能：

undefined是当前组件没有子节点；

object是有一个子节点

array是有多个子节点。

解决方法：用React.Children来处理this.props.children.可以用React.Children.map遍历子节点，所以不用担心this.props.children的数据类型。

Demo06: PropTypes（验证组件实例属性是否符合要求）/defaultProps(设置属性的默认值)

​ 下面的代码中MyTitle组件有一个title属性。propTypes告诉React,这个title是必须的，而且它的值必须是字符串。

class MyTitle extends React.Component {
  static propTypes = {
    title: PropTypes.string.isRequired,
  }
  render() {
    return <h1> {this.props.title} </h1>;
  }
}

​ 下面的代码，我设置title属性是数值，报错，原因如上。

var data = 123;

ReactDOM.render(
  <MyTitle title={data} />,
  document.getElementById('example')
);

​ 控制台输出的错误信息如下

Warning: Failed propType: Invalid prop `title` of type `number` supplied to `MyTitle`, expected `string`.

更多的PropTypes设置，可以查看React官方文档.

defaultProps方法用来设置组件属性的默认值，下面的代码输出：Hello World

class MyTitle extends React.Component {
  constructor(props) {
    super(props)
  }
  static defaultProps = {
    title: 'Hello World',
  }
  render() {
    return <h1> {this.props.title} </h1>;
  }
}

ReactDOM.render(
  <MyTitle />,
  document.getElementById('example')
);

Demo07: Finding a DOM node(获取真实DOM节点)

用到 ref 属性，,然后this.refs.[refName]返回真实DOM节点。必须等到DOM插入文档后才能使用，否则会返回空值报错。

所以通过为组件指定Click回调函数，确保等到真实DOM发生Click事件后，才会读取this.refs.[refName]属性。注意：属性事件创建React.createClass.

第二种属性事件创建是React.createRef()，运用了继承，和c++/java类似。

/*第一种写法*/
/*
var MyComponent = React.createClass({
    handleClick:function(){
        this.refs.myTextInput.focus();
    },
    render: function(){
        return{
            <div>
                <input type="text" ref="myTextInput"/>
                <input type="button" value="Focus the text input" onClick={this.handleClick}/>
            </div>
        }
    }
})
*/
/*第二种写法*/
class MyComponent extends React.Component {
  constructor(props) {
    super(props);
    this.myTextInput = React.createRef();
    this.handleClick = this.handleClick.bind(this)
  }
  handleClick() {
    this.myTextInput.current.focus();
  }
  render() {
    return (
      <div>
        <input type="text" ref={this.myTextInput} />
        <input type="button" value="Focus the text input" onClick={this.handleClick} />
      </div>
    );
  }
}

ReactDOM.render(
  <MyComponent />,
  document.getElementById('example')
);

方法一getInitalState定义初始化状态。

方法二this.state 定义初始化状态，bind是重定向this指向,this.setState()更新 this.state 然后重新渲染UI

/*第一种方法*/
var LikeButton = React.createClass({
    getInitalState: function(){
        return {liked: false};
    },
    *****
})
/*第二种方法*/
class LikeButton extends React.Component {
  constructor(props) {
    super(props)
    this.state = {
        liked: false
    }
    this.handleClick = this.handleClick.bind(this)
  }
   /*相同部分如下*/ 
  handleClick(event) {
    this.setState({ liked: !this.state.liked });
  }
  render() {
    var text = this.state.liked ? 'like' : 'haven\'t liked';
    return (
      <p onClick={this.handleClick}>
        You {text} this. Click to toggle.
      </p>

  }

Demo09: Form(表单)

用户输入的内容，属于用户跟组件互动，不能用this.props读取。

value 表单属性，比如<input>, <textarea>, and <option>，/select,radio/,用onChange事件.

下面代码，方法一：

文本输入框的值，不用用this.props.value读取，要定义一个onChange事件回调函数，通过event.target.value读取用户输入值。

/*方法一*/
var Input = React.createClass({
    getInitialState: function(){
        return {value:'Hello!'}
    },
    ******
})
/*方法二*/
class Input extends React.Component {
constructor(props) {
  super(props)
  this.state = {value: 'Hello!'}
  this.handleChange = this.handleChange.bind(this)
}
    /*下面相同*/
handleChange(event) {
  this.setState({value: event.target.value});
}
render() {
  var value = this.state.value;
  return (
    <div>
      <input type="text" value={value} onChange={this.handleChange} />
      <p>{value}</p>
    </div>
  );
}
}

ReactDOM.render(<Input/>, document.getElementById('example'));

Demo10: Component Lifecycle(组件生命周期)

组件生命周期分成三个状态：

  • Mouting:已插入真实DOM

  • Updating:正在被重新渲染

  • Unmounting:已移出真实DOM

    React为每个状态都提供了两种处理函数，will函数在进入状态前调用，did函数在进入状态后调用，三种状态共计5种处理函数。

    • componentWillMount()：插入真实DOM前

    • componentDidMount():插入真实DOM后

    • componentWillUpdate(object nextProps, object nextState):重新渲染前

    • componentDidUpdate(object prevProps, object prevState):重新渲染后

    • componentWillUnmount():移除真实DOM前

      此外，React还提供两种特殊状态处理函数。

      • componentWillReceiveProps(object nextProps):已加载组件受到新的参数时调用
      • shouldComponentUpdate(object nextProps,object nextState):组件判断是否重新渲染时调用
/*第一种*/
var Hello = React.createClass({
    getInitialState: function(){
        return {opacity: 1.0}
    };
    ******
})
/*第二种*/
class Hello extends React.Component {
  constructor(props) {
    super(props)
    this.state = {opacity: 1.0};
  }
/*同理一样*/
  componentDidMount() {
    this.timer = setInterval(function () {
      var opacity = this.state.opacity;
      opacity -= .05;
      if (opacity < 0.1) {
        opacity = 1.0;
      }
      this.setState({
        opacity: opacity
      });
    }.bind(this), 100);
  }

  render() {
    return (
      <div style={{opacity: this.state.opacity}}>
        Hello {this.props.name}
      </div>
    );
  }
}

ReactDOM.render(
  <Hello name="world"/>,
  document.getElementById('example')
);

上面代码中，在Hello组件加载后，通过componentDidMount方法设置一个定时器，每个100ms，就重新设置组件透明度，从而重新渲染。

注意：style属性设置方式不能写成

style="opacity: {this.state.opacity};"

而要写成

style={{opacity: this.state.opacity}}

原因：React组件样式是一个对象，所以第一种大括号是js语法，第二重大括号表示样式对象。

官网文档如下：

  • componentWillMount(): Fired once, before initial rendering occurs. Good place to wire-up message listeners. this.setState doesn’t work here.
  • componentDidMount(): Fired once, after initial rendering occurs. Can use this.getDOMNode().
  • componentWillUpdate(object nextProps, object nextState): Fired after the component’s updates are made to the DOM. Can use this.getDOMNode() for updates.
  • componentDidUpdate(object prevProps, object prevState): Invoked immediately after the component’s updates are flushed to the DOM. This method is not called for the initial render. Use this as an opportunity to operate on the DOM when the component has been updated.
  • componentWillUnmount(): Fired immediately before a component is unmounted from the DOM. Good place to remove message listeners or general clean up.
  • componentWillReceiveProps(object nextProps): Fired when a component is receiving new props. You might want to this.setState depending on the props.
  • shouldComponentUpdate(object nextProps, object nextState): Fired before rendering when new props or state are received. return false if you know an update isn’t needed.

var UserGist = React.createClass({

class UserGist extends React.Component {

Demo12: Display value from a Promise

可以把Promise对象传入组件，下面把Promise对象作为属性，传给RepoList组件。

ReactDOM.render(
  <RepoList promise={$.getJSON('https://api.github.com/***?name=shirley&password=***')} />,
  document.getElementById('example')
);

如果Promise对象正在抓取数据（pending状态）,组件显示loading(正在加载)；

如果Promise对象报错（rejected状态）,组件显示error ***(报错信息)；

如果Promise对象抓取数据成功（fullfilled)状态，组件显示获取到的数据。

class RepoList extends React.Component {
  constructor(props) {
    super(props)
    this.state = {
      loading: true,
      error: null,
      data: null
    };
  }

  componentDidMount() {
    this.props.promise.then(
      value => this.setState({loading: false, data: value}),
      error => this.setState({loading: false, error: error}));
  }

  render() {
    if (this.state.loading) {
      return <span>Loading...</span>;
    }
    else if (this.state.error !== null) {
      return <span>Error: {this.state.error.message}</span>;
    }
    else {
      var repos = this.state.data.items;
      var repoList = repos.map(function (repo, index) {
        return (
          <li key={index}><a href={repo.html_url}>{repo.name}</a> ({repo.stargazers_count} stars) <br/> {repo.description}</li>
        );
      });
      return (
        <main>
          <h1>Most Popular JavaScript Projects in Github</h1>
          <ol>{repoList}</ol>
        </main>
      );
    }
  }
}
