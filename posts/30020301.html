2.vue生命周期函数详解

​ vue生命周期分为四个阶段

组件创建时（creating):beforeCreate,created

模板渲染时（mounting):beforeMount,mounted

数据更新时（updating):beforeUpdate,updated

组件卸载时（destroying):beforeDestory,destoryed

4.axios是什么？怎么使用？描述使用它实现登录功能的流程？

1.axios是基于promise，用于浏览器和nodejs的一个http客户端主要用于向后台发起请求的，还有就是在请求中做更多的控制。

2.优势：支持promise;提供了一些并发的方法；提供拦截器；提供支持CSRF（跨站请求伪造）

3.axios,fetch,ajax(jquery)区别

前两者都是支持promise语法，后者默认使用callback方式

fetch 本质上脱离的xhr 是新的语法（有自己的特性 默认不传cookie 不能像xhr这样 去监听请求的进度）

4.2拦截登录代码

//http request 拦截器
axios.interceptors.request.use(
    config=>{
        if(store.state.token){//判断是否存在token,如果存在的话，则每个http header都加上token
            config.headers.Authorization = `token ${store.state.token}`;
        }
        return config;
    },
    err=>{
        return Promise.reject(err);
    }
);
//http response 拦截器
axios.interceptors.response.use(
    response=>{
        return response;
    },
    error=>{
        if(error.response){
            switch(error.response.status){
                case 401:
                    //返回401 清除token信息并跳转到登录页面
                    store.commit(types.LOGOUT);
                    router.replace({
                        path:'login',
                        query:{redirect:router.currentRoute.fullPath}
                    })
            }
        }
       return Promise.reject(error.response.data)//返回接口返回的错误信息 
    } 
);

5.vuex是什么？怎么使用？哪种功能场景使用它？

5.1关于vuex

vuex 是一个专门为vue构建的状态集管理

主要是为了解决组件间状态共享的问题

强调的是集中式管理

说白了 主要是便于维护 便于解耦 所以不是所有的项目都适合使用vuex

如果不是构建大型项目 使用vuex 反而使项目代码繁琐多余

5.2vuex的核心

state mutations getters actions modules

1.存放数据的（必须） 2.更改状态的(必须) 3.类似于computed 4. 更改状态的（必须）5.拆分仓库的，导出模块

5.3vuex使用案例

<script src="js/vue.js"></script>
<script src="js/vuex.js"></script>
<div id="app">
    <p>
        更新计数{{count}}
    </p>
    <button @click="add">
        ++
    </button>
    <button @click="reduce">
        --
    </button>
</div>
<script>
    var store = new Vuex.Store({
        state:{
            count:0
        },
        mutations:{//唯一一个可以更改状态的地方
            addState:function(state){
                state.count++;
            },
            reduceState:function(state){
                state.count--;
            }
        }
    })
    var vm = new Vue({
        el:"#app",
        store,
        computed:{
           count:function(){
               return this.$store.state.count;
           }
        },
        methods:{
            add:function(){
                store.commit("addState");
            },
            reduce:function(){
                store.commit("reduceState");
            }
        }
        /*data:{
            count:0
        },
        methods:{
            add:function(){
                this.count++;
            },
            reduce:function(){
                this.count--;
            }
        }*/ 
    })
</script>

7.导航钩子有哪些？它们有哪些参数？

​ 导航钩子 翻译过来就是路由的生命周期函数（vue-router)

可用于路由拦截器

它其实主要分为两种 全局和局部

全局的钩子函数：

beforeEach:在路由切换开始时调用，

afterEach:在路由切换离开时调用，

局部到单个路由：beforeEnter;

组件的钩子函数：

beforeRouterEnter,

beforeRouterUupdate,

beforeRouterLeave

to:即将进入的目标对象

from:当前导航要离开的导航对象

next:是一个函数 调用resolve 执行下一步

7.1钩子函数使用案例

<script>
    const router = new VueRouter({
        routes:routes
    });
    const routes=[
        {
            path:'/',
            name:'/',
            component:Home,
            beforeEnter:function(){

            }
        },
        {
            path:'/login',
            name:'login',
            component:Login
        }，
        {
            path:'/buy',
            name:'buy',
            component:Buy,
            meta:{
                auth:true
            }
        }
    ];
    //全局路由
    router.beforeEach((to,from.next)=>{
        if(to.meta.auth){
            if(token){
                next()
            }else{
                next({
                    path:'/login',

                })
            }
        }
    })
</script>

8.vue-model是什么？Vue中标签怎么绑定事件？

vue中利用v-model来进行表单数据的双向绑定

v-bind 绑定了一个value属性，v-on 把当前的元素绑定到了一个事件上

13.对vue中虚拟DOM的理解

什么是虚拟dom？

说白了 就是以js对象的形式 去添加dom元素

本质上是优化了diff算法

采用了 新旧dom的对比 获取差异的dom 一次性更新到真实的dom上

虚拟dom本身也有自己的缺陷，它更适合批量修改dom

尽量不要跨层级修改dom

设置key 可以最大的利用节点

DOM节点在HTML文档中的表现通常是这样的
<ul id="list">
    <li>Item 1</li>
    <li>Item 2</li>
</ul>
DOM节点也可以表示为一个JavaScript对象，就像这样

    tag:'ul',
    attributes:{
        id:'list'

        //这里是li

};

14.对vue的MVVM理解

如何理解Vue的Mvvm模式？

mvvm 和 mvc 之间的区别

mvc 是 model view controller

view<==>controller==>model<==>view

mvc流程：用户输入-》Controller控制器-》Model模型-》View视图-》用户获得反馈；Model模型和数据库交互

mvvm 也分为三层model view viewModel

而vue就是专注于view和viewModel的框架

15.Vue中的keep-alive的作用

什么是keep-alive?

说白了 它能上不活动的组件 “活着”

它提供了include与exclude两个属性 允许组件有条件的缓存

实现的原理是什么？

其实就是在created 时候 将需要缓存的vnode节点 放到cache中 在render的时候 根据name再进行取出

怎么去使用它？

//routes/index.js
routes:[

    },
